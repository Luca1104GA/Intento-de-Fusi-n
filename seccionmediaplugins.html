Complementos HTML

Los     Plug-ins    son programas informáticos que amplían la funcionalidad estándar del navegador.

Plug-ins

Estos complementos se diseñaron para usarse con muchos propósitos deferentes:

  • Para ejecutar subprogramas de Java
  • Para ejecutar controles Microsoft ActiveX
  • Para visualizar películas Flash
  • Para mostrar mapas
  • Para buscar virus
  • Para verificar una identificación bancaria

Advertencia !

La mayoria de los navegadores ya no admiten subprogramas ni complemetentos de Java.
Los controles ActiveX ya no son compatibles con ningún navegador.
La compatiblidad con Shockwave Flash también se ha desactivado en los navegadores modernos.

<object>

El elemento <object> es compatible con todos los navegadores.
Este elemento define un objeto incrustado dentro de un documento HTML.

Fue diseñado para incrustar Plug-ins  (como subprogramas de Java, lectores de PDF y reproductores Flash)
en páginas web, pero también se puede utilizar para incluir HTML en HTML.

Ejemplo

<object width="100%" height="500px" data="snippet.html"></object>

O imagenes si lo deseas:

<object data="flowers.jpeg"></object>

<embed>

El elemento <embed> es compatible con todos los principales navegadores.
Este elemento también define un objeto incrustado dentro de un documento HTML.

Los navegadores web admiten el elemento <embed> desde hace mucho tiempo.
Sim embargo, no formaba parte de la especificación HTML antes de HTML5.

Ejemplo

<embed src="smallflowers.jpeg">

Tenga en cuenta que el elemento <embed> no tiene una etiqueta de cierre.
No puede contener texto alternativo.

El elemento <embed> también se puede utilizar para incluir HTML en HTML

<embed width="100%" height="500px" src="snippet.html">

Resumen:

  • <object> es más versátil y se utiliza para una gama más amplia de tipos de contenido.
  • <embed> está diseñado específicamente para contenido multimedia, como audio y video.
  • Ambos elementos tienen atributos para especificar la fuente ('data' o 'src')
    y el tipo MIME ('type') del contenido.
  • <object> Permite agregar contenido de respaldo, mientras que <embed> no.
  • La elección entre ellos depende del tipo de contenido que estés incrustando y de tus preferencias de desarrollo.
    En general, se tiende a preferir el uso de <object> para mayor flexibilidad
    y capacidad de manejar una variedad de contenido mas amplia.
