API de trabajadores web HTML

Un trabajador web es un JavaScript que se ejecuta en segundo plano, sin afectar el rendimiento de la página.

¿Qué es un trabajador web?

Al ejecutar secuencias de comandos en una página HTML, la página deja de responder hasta que finaliza la secuencia de comandos.

Un trabajador web (web worker) es un JavaScript que se ejecuta en segundo plano, independiente de otros scripts, sin afectar el rendimiento de la página.
Puedes continuar haciendo lo que quiertas : hacer click, seleccionar cosas, etc., mientras el trabajador web se ejecuta en segundo plano.

Consultar soporte para Web Workers

Antes de crear un Web Worker, verifique si el navegador del usuario lo admite:

if (typeof(Worker) !== "undefined") {
  // Si! Web worker support!
   // Algo de Código.....
 } else {
  // Sorry! No Web Worker support..
 }

Crear un archivo Web Worker

Ahora, creemos nuestro Web Worker en un JavaScript externo.
Aquí, creamos un script que cuenta. El script se almacena en el archivo "demo_workers.js":

var i = 0;

function timedCount() {
  i = i + 1;
  postMessage(i);
  setTimeout("timedCount()",500);
}

timedCount();

La parte importante del código anterior en el método postMessage(), que se utiliza para publicar un mensaje en la página HTML.

Nota: Normalmente, los Web Workers no se utilizan para scripts tan simples, sino para tareas que requieran más uso de CPU.

Crear un Web Worker Object

Ahora que tenemos el archivo Web Worker, debemos llamarlo desde una página HTML.

Las siguientes líneas verifican si el Worker ya existe; de lo contrario, crea un nuevo Web Worker Object y ejecuta el código en "demo_workers.js":

if (typeof(w) == "undefined") {
  w = new Worker("demo_workers.js");
}

Luego podremos enviar y recibir mensajes del Web Worker.

Agregue un detector de eventos "onmessage" al Web Worker.

w.onmessage = function(event){
  document.getElementById("result").innerHTML = event.data;
};

Cuando el Web Worker publica un mensaje, se ejecuta el código dentro del detector de eventos.
Los datos del Web Worker se almacenan en event.data;

Terminar un Web Worker

Cuando se crea un Web Worker Object, continuará escuchando mensajes (incluso después de que finalice el script externo ) hasta que finalice.

Para finalizar un Web Worker y liberar recursos del navegador/computadora, utilice el método terminate() :

w.terminate();

Reutilizar el Web Worker

Si configura la variable de trabajo como indefinida, una vez terminada, puede reutilizar el código:

w = undefined;

Código de ejemplo Web Worker completo

Ya hemos visto el código del Worker en el archivo .js.
Acontinuación se muestra el código de la página HTML:

<!DOCTYPE html>
<html>
<body>

<p>Count numbers: <output id="result"></output></p>
<button onclick="startWorker()">Start Worker</button>
<button onclick="stopWorker()">Stop Worker</button>

<script>
var w;

function startWorker() {
  if (typeof(Worker) !== "undefined") {
    if (typeof(w) == "undefined") {
      w = new Worker("demo_workers.js");
    }
    w.onmessage = function(event) {
      document.getElementById("result").innerHTML = event.data;
    };
  } else {
    document.getElementById("result").innerHTML = "Sorry! No Web Worker support.";
  }
}

function stopWorker() {
  w.terminate();
  w = undefined;
}
</script>

</body>
</html>

Web Worker y el DOM

Dado que los Web Workers están en archivos externos, no tienen acceso a los siguientes Objects JavaScript:

  • El window object
  • El document object
  • El parent object
