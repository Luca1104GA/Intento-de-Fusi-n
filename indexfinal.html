Diseño web adaptable

El diseño web responsivo consiste en crear páginas web que se vean bien en todos los dispositivos.
Un diseño web responsivo se ajustará automaticamente a diferentes tamaños de pantalla y ventanas gráficas.

¿Qué es el diseño web responsivo?

El diseño web responsivo consiste en utilizar HTML y CSS para cambiar el tamaño, ocultar, reducir o
ampliar automáticamente un sitio web, para que se vea bien en todos los dispositivos.
(Computadoras de escritorio, tablets y celulares)

Configuración de la ventana gráfica (Viewport)

Para crear un sitio web responsivo, agregue la etiqueta <meta> a todas sus páginas web:

Ejemplo

<meta name="viewport" content="width=device-width, initial-scale=1.0">

Esto configurará la ventana gráfica de su página, que le dará al navegador instruciones sobre cómo
controlar las dimensiones y la escala de la página.

Imágenes responsivas

Las imágenes responsivas son imágenes que se escalan bien para adaptarse a cualquier tamaño de navegador.

Usando la propiedad de ancho

Si la proiedad width se establece en 100%, la imagen responderá y se ampliara hacia arriba y abajo :

Ejemplo

<img src="img_girl.jpg" style="width:100%;">

Observe que en el ejemplo anterior, la imagen se puede ampliar para que sea mayor que su tamaño original.
Una mejor solución, en muchos casos será utilizar la proiedad max-width en su lugar.

Usando la propiedad ancho máximo (max-width)

Si la propiedad max-width se establece en 100%, la imagen se reducirá si es necesario,
pero nunca se ampliará para que sea mayor que su tamaño original:

Ejemplo

<img src="img_girl.jpg style="max-width:100%;height:auto;">

Mostrar diferentes imágenes según el ancho del navegador

El elemento <picture> le permite definir diferentes imágenes para diferentes tamaños del navegador.

Cambie el tamaño de la ventana del navegador para ver cómo la imagen a continuación cambia según el ancho:

<picture> <source srcset="img_smallflower.jpg" media="(max-width: 600px)> <source srcset="img_flowers.jpg" media="(max-width: 1500px)> <source srcset="flowers.jpg"> <img src="img_smallflower.jpg alt="Flowers"> </picture>

Tamaño de texto adaptable

El tamaño del texto se puede configurar con una unidad "vw", que significa "viewport width" (ancho de ventana gráfica)
De esa forma, el tamaño del texto seguirá el tamaño de la ventana del navegador:

<h1 style="font-size:10vw">Hola Mundo!</h1>

Hola Mundo!

Cambie el tamaño de la ventana del navegador para ver cómo cambia el tamaño del texto.

Viewport se refiere al tamaño de la ventana del navegador. 1vw representa el 1% del ancho del viewport.
Si el viewport tiene un ancho de 50 cm, entonces 1vw equivaldria a 0,5 cm. En otras palabras,
vw es una unidad de medida relativa que se adapta al tamaño de la ventana del navegador,
lo que permite crear diseños web responsivos.

Elementos del código de programación

HTML contiene varios elementos para definir la entrada del susuario y el código de programación.

<kbd> para entrada de teclado

El elemento HTML <kbd> se utiliza para mostrar la entrada del teclado. El contenido interior
se muestra en la fuente monoespaciada predeterminada del navegador.

Ejemplo

Defina algo de texto como entrada de teclado en un documento:

         <p> Guarde el documento presionando <kbd>Ctrl + S<kbd> </p>

Resultado:

Guarde el documento presionando Ctrl + S

<samp> Para el Output

El elemento HTML <samp> se utiliza para mostrar un ejemplo Output de un codigo de programación.
El contenido interior se muestra en la fuente monoespaciada predeterminada del navegador.

"La etiqueta HTML <samp> se utiliza para mostrar ejemplos de salida de programas o
fragmentos de código en una página web. Proporciona una forma de destacar y formatear
este texto de salida, lo que lo hace más fácil de identificar en el contexto de la página."

Ejemplo

Define algo de texto como una muestra Output de un codigo de programación.

<p>Mensaje desde mi computadora:</p> <p><samp>File not found.<br>Press F1 to continue</samp></p>

Resultado:

Mensaje desde mi computadora:

File not found.
Press F1 to continue

<code> Para codigos de programación

El elemento HTML <code> se utiliza para definir una parte de un codigo de programación.
El contenido interior se muestra en la fuente monoespaciada predeterminada del navegador.

"La etiqueta HTML <code> se utiliza para resaltar y formatear fragmentos de código en una página web.
Al utilizar esta etiqueta, el texto contenido dentro de <code> se muestra en una fuente de ancho fijo,
como una fuente de máquina de escribir, y suele estar rodeado por un borde o fondo distintivo para diferenciarlo del texto normal.
Esto es especialmente útil cuando deseas mostrar ejemplos de código fuente, comandos de programación
o cualquier otro contenido relacionado con la codificación en tu página web.
La etiqueta <code> ayuda a mantener la estructura y la legibilidad del código dentro del contexto de tu página."

Ejemplo:

Define algo de codigo de programación como texto en un documento:

            <code> x = 5; y = 6; z = x + y; </code

Resultado:

x = 5; y = 6; z = x + y;

Note como el elemento <code> no preserva espacios en blanco extra y saltos de linea.
Para arreglar esto, puedes poner el <code> adentro de un elemento <pre>

            <pre>             <code> x = 5; y = 6; z = x + y; </code             </pre>

Resultado:

x = 5;
y = 6;
z = x + y;

"La etiqueta <pre> en HTML se utiliza para definir texto preformateado,
Lo que significa que el texto dentro de esta etiqueta se muestra tal como está escrita en el código fuente,
incluyendo espacion en blanco y saltos de linea. Esta etiqueta es útil cuando deseas mostrar código fuete,
bloques de texto con un formato específico, o cualquier contenido donde la disposición exacta de los caracteres es importante. "

<var> Para variables

El elemento HTML <var> se utiliza para definir una variable en programación o en una Expresión matemática.
Su contenido normalmente se muestra en cursiva.

Ejemplo

Defina algo de texto como variables en un documento:

<p>
   El area de un triangulo es : 1/2 x <var>b</var x <var>h</var, donde <var>b</var> es la base,
   y <var>h</var> es la altura vertical.
</p>

Resultado:

El area de un triangulo es : 1/2 x b x h, donde b es la base, y h es la altura vertical.

"<var> se utiliza para representar una variable o una expresión matemática en un contexto de texto.
Esta etiqueta se utiliza para indicar que un fragmento de texto dentro de una página web representa una variable o una expresión matemática.
Por lo general, el texto dentro de <var> se muestra en cursiva para distinguirlo del texto circundante.

Es útil en situaciones donde deseas resaltar que ciertos elementos en tu texto son variables, como en explicaciones matemáticas o cientificas.

Resumen del capítulo

  • <kbd> define una entrada de teclado
  • <samp> define una salida de muestra de un programa de computadora
  • <code> define un fragmento de código de programación
  • <var> define una variable en programación o en una expresión matemática
  • <pre> define texto preformateado

Elementos semánticos

Elementos semánticos = Elementos con significado.

¿Qué son los elementos semánticos?

Un elemento semántico describe claramente su significado tanto para el navegador como para el desarrolador.

  • Ejemplos de elementos no semánticos: <div> y <span> - No dicen nada sobre su contenido.
  • Ejemplos de elementos semánticos: <form>, <table>, y <article> - Definen claramente su contenido.

con la tecla de tabulación pueden variar en anchura dependiendo de la configuración de la fuente o el editor,
lo que puede causar inconsistencias en el formato del código cuando se comparte con otros desarrolladores o se visualiza
en diferentes entornos. Usar dos espacios en blanco como sangría asegura una mayor consistencia.

Configuración de la ventana gráfica (Viewport)

El Viewport es el área visible para el usuario de una página web. Varía según el dispositivo,
sera más pequeño en un teléfono móvil que en la pantalla de una computadora.

Debes incluir el siguiente elemento <meta> en todas tus páginas web:

<meta name="viewport" content="width=device-width, initial-scale=1.0">

Esto le da al navegador instrucciones sobre cómo controlar las dimensiones y la escala de la página.

La parte width=device-width establece el ancho de la página para seguir el ancho de la pantalla
del dispositivo (que variará según el dispositivo).

La parte initial-scale=1.0 establece el nivel de zoom inicial cuando el navegador carga la página por primera vez.

Extensiones de archivo

  • Los archivos HTML deben tener una extensión .html (se permite .htm)
  • Los archivos CSS deben tener una extensión .css
  • Los archivos javaScript deben tener una extensión .js

¿Diferencias entre .htm y .html?

¡No hay diferencia entre las extenciones de archivo .htm y .html!
Ambos seran tratados como HTML por cualquier navegador y servidor web.

La principal diferencia entre .htm y .html radica simplemente en la longitud de la extensión del archivo.
Históricamente, algunas versiones más antiguas de sistemas operativos o servidores tenían restricciones en
la longitud de las extensiones de archivo, por lo que se utilizaba ".htm " en lugar de ".html" para ahorrar caracteres.
Sin embargo, con el tiempo y la evolución de los sistemas, esta limitación se volvió menos relevante, y ".html" se
convirtió en la extensión más comúnmente utilizada.

Hoy en día, ambas extensiones, htm y .html, son ampliamente reconocidas y aceptadas por
los navegadores web y los servidores web. Puedes usar cualquiera de las dos extensiones
para tus archivos HTML sin preocuparte por problemas de compatibilidad. Sin embargo,
".html" se ha vuelto más común y se recomienda como una práctica estándar en la mayoria de los casos.

En resumen, no hay una diferencia funcional real entre .htm y .html; ambos son tratados
como archivos HTML y funcionarán de manera similar en los navegadores web y servidores web modernos.

Nombres de archivos predeterminados

Cuando una URL no especifica un nombre de archivo al final (como "https://www.w3schools.com/"), el servidor simplemente
agrega un nombre de archivo predeterminado, como "index.html","index.htm","default.html" o "default.htm".

Si su servidor está configurado sólo con "index.html" como nombre de archivo predeterminado,
su archivo debe llamarse "index.html" y no "default.html".

Sin embargo, los servidores se pueden configurar con más de un nombre de archivo predeterminado;
normalmente puedes configurar tantos nombres de archivos predeterminados como quieras.

Espacio no rompible

Una entidad HTML comúnmente utilizada es el espacio no rompible(&nbsp;).
Un espacio no rompible es un espacio que no se dividirá en una nueva línea.

Dos palabras separadas por un espácio no rompible se mantendrán juntas (no se dividirán en una
nueva línea). Esto es útil cuando dividir las palabras podría resultar disruptivo.

Ejemplos:

  • 10 km/h
  • 10 PM

Otro uso común de los espacios no rompibles es evitar que los navegadores trunquen espacios en las páginas HTML.

Si escribe 10 espacios en su texto, el navegador eliminará 9 de ellos. Para agregar espacios reales a su texto
puedes usar la entidad caracter &nbsp.

El espacio no rompible "guión" (&#8209;) es usado para definir un caracter de guión (‑) y no se rompe en una nueva línea.

ISO-8859-1 era el juego de caracteres predeterminado para HTML 4.

Localizadores uniformes de recursos HTML (URL)

Una URL es otra palabra para una dirección web.

Una URL puede estar compuesta de palabras (por ejemplo, youtube.com) o una dirección de
Protocolo de internet (IP) (por ejemplo: 192.68.20.50).

La mayoria de las personas ingresan el nombre cuando navegan, porque los nombres son más
fáciles de recordar que los números.

URL

Los navegadores web solicitan páginas de servidores web mediante una URL.

Un localizador uniforme de recursos (URL) se utiliza para dirigirse a un documento (u otros datos) en la web.

scheme://prefix.domain:port/path/filename
  • scheme: define el tipo de servicio de Internet (el más común es http o https)
  • prefix: define un prefijo de dominio (el valor predeterminado para http es WWW)
  • domain: define el nombre de dominio de Internet (como youtube.com)
  • port: define el número de puerto en el host (el valor predeterminado para http es 80)
  • path: define una ruta en el servidor (si se omite: el directorio raíz del sitio)
  • filename: define el nombre de un documento o recurso

Codificación de URL

Las URL solo se pueden enviar a través de Internet utilizando el character set ASCII.
Si una URL contiene caracteres fuera del conjunto ASCII, la URL debe convertirse.

La codificación de URL convierte caracteres que no son ASCII a un formato que se puede transmitir a través de Internet.
La codificación de URL reemplaza a los caracteres que no son ASCII con un "%" seguido de dígitos hexadecimales.

Las URL no pueden contener espacios. La codificación de URL normalmente reemplaza un espacio con un signo más (+), o %20.

Inténtalo tú mismo

Si hace click en "Enviar". el navegador codificará la URL de la entrada antes de enviarla al servidor.
Una página en el servidor mostrará la entrada recibida.

HTML vs XHTML

XHTML es una versión de HTML más estricta y más basade en XML.

¿Qué es XHTML?

  • XHTML Siglas de EXtensible HyperText Markup Language (lenguaje de marcado de hipertexto extensible)
  • XHTML es una versión más estricta y más basada en XML
  • XHTML es HTML definido como una aplicación XML
  • XHTML es compatible con todos los principales navegadores

¿Por qué XHTML?

XML es un lenguaje de marcado en el que todos los documentos deben marcarse correctamente (estar "bien formados").

XHTML se desarrolló para hacer que HTML sea más extensible y flexible para trabajar con otros formatos de datos (como XML).
Además, los navegadores ignoran los errores en las páginas HTML e intentan mostrar el sitio web incluso si tiene algunos
errores en el marcado. Por lo tanto, XHTML viene con un manejo de errores mucho más estricto.

Si desea estudiar XML lea la Guia tutorial de XML

Las diferencias más importantes con respecto a HTML

  • <!DOCTYPE> es obligatorio
  • El atributo xmlns en <html> es obligatorio
  • <html>, <head>, <title> y <body> son obligatorios
  • Los elementos siempre deben estar correctamente anidados
  • Los elementos siempre deben estar cerrados
  • Los elementos siempre deben estar en minúsculas
  • Los nombres de los atributos siempre deben estar en minúsculas
  • Los valores de los atributos siempre deben citarse
  • La minimización de atributos está prohibida

XHTML - <!DOCTYPE...> es obligatorio

Un documento XHTML debe tener una declaración XHTML <!DOCTYPE>

Los elementos <html>, <head>, <title> y <body> también deben estar presentes,
y el atributo xmlns en <html> debe especificar el espacio de nombres xml para el documento.

Aquí hay un documento XHTML con un mínimo de etiquetas requeridas:

</head>

  Algo de contenido aquí...

</body>
</html>

En XHTML, los elementos siempre deben estar correctamente anidados entre sí, así:

Correcto:
<b><i>Algo de texto</i></b>
Equivocado:
<b><i>Algo de texto</b></i>

<p>Esto es un párrafo
<p>Esto es otro párrafo

En XHTML, los elementos vacíos siempre deben estar cerrados, así:

Correcto:
Un salto de línea: <br />
Una regla horizontal: <hr />
Una imagen: <img src="happy.gif" alt="Happy face" />
Equivocado:
Un salto de línea: <br>
Una regla horizontal: <hr>
Una imagen: <img src="happy.gif" alt="Happy face">

En XHTML, los valores de los atributos siempre deben ir entre comillas, así:

Equivocado:
<a href=https://www.w3schools.com/html/>Visit our HTML tutorial</a>

<input type="checkbox" name="vehicle" value="car" checked="checked" />

Atributos de form

Este capítulo describe los diferente atributos del elemento <form>

El atributo action

El tributo action define la acción que se realizará cuando se envíe el formulario.

Por lo general, los datos del formulario se envian a un archivo en el
servidor cuando el usuario hace click en el botón enviar.

En el siguiente ejemplo, los datos del formulario se envían a un archivo llamado "action_page.php".
Este archivo contiene un script del lado del servidor que maneja los datos del formulario.

Ejemplo

Al presionar, se envian los datos del formulario a "action_page.php"






<form action="/action_page.php">
  <label for="fname">First name:</label><br>
  <input type="text" id="fname" name="fname" value="John"><br>
  <label for="lname">Last name:</label><br>
  <input type="text" id="lname" name="lname" value="Doe"><br><br>
  <input type="submit" value="Submit">
</form>

Consejo: se se omite el atributo action, la acción se establece en la página actual.

El atributo target

El atributo target especifica dónde mostrar la respuesta que se recibe después de enviar el formulario.

El atributo target puede tener uno de los siguientes valores:

Valor Descripcion
_blank La respuesta se muestra en una nueva ventana o pestaña
_self La respuesta se muestra en la ventana actual
_parent La respuesta se muestra en el marco principal (frame) padre
_top La respuesta se muestra en todo el cuerpo de la ventana
framename La respuesta se muestra en un iframe nombrado

El valor predeterminado es _self lo que significa que la respuesta se abrirá en la ventana actual.

Ejemplo

Aquí el resultado enviado se abrirá en una nueva pestaña del navegador:

<form action="/action_page.php" target="_blank">

El atributo method

El atributo method especifica el método HTTP que se utilizará al enviar los datos del formulario.

Los datos del formulario se pueden enviar como variables de URL (con method="get") o
como transacción posterior HTTP (con method="post").

El método HTTP predeterminado al enviar datos de formulario es GET.

Ejemplo

Este ejemplo utiliza el método GET al enviar los datos del formulario:

<form action="/action_page.php" method="get">
Ejemplo

Este ejemplo utiliza el método POST al enviar los datos del formulario:

<form action="/action_page.php" method="post">
Notas sobre GET:
  • Agrega los datos del formulario a la URL, en pares de nombre/valor
  • ¡NUNCA utilice GET para enviar datos confidenciales! (¡Los datos del formulario enviado son visibles en la URL!)
  • La longitud de una URL es limitada (2048 caracteres)
  • Útil para envíos de formularios donde un usuario desea marcar el resultado como favorito
  • GET es buerno para datos no seguros, como cadenas de consulta en Google
Notas sobre POST
  • Agrega los datos del formulario dentro del cuerpo de la solicitud HTTP (los datos del formulario enviado no se muestran en la URL)
  • POST no tiene limitaciones de tamaño y puede usarse para enviar grandes cantidades de datos
  • Los envíos de formularios con POST no se pueden marcar como favoritos

consejo: ¡Utilice siempre POST si los datos del formulario contienen información personal o confidencial!

El atributo autocomplete

Cuando la función de autocompletar está activada, el navegador completa automáticamente los valores
en función de los valores que el usuario ha ingresado anteriormente.

Ejemplo

Un formulario con autocompletar:



<form action="/action_page.php" autocomplete="on">

Notas sobre el metodo "get":
  • Este método agrega los datos del formulario a la URL en pares de nombre/valor.
  • Este método es útil para envíos de formularios en los que un usuario desea marcar el resultado como favorito.

  • ¡Nunca utilice el método "get" para pasar información confidencial! (La contraseña u otra
    información confidencial será visible en la barra de direcciones del navegador)
Notas sobre el método "post":
  • Este método envía los datos del formulario como una transacción posterior HTTP.
  • Los envíos de formularios con el método "post" no se pueden marcar como favoritos.
  • El método "post" es más robusto y seguro que "get" y "post" no tiene limitaciones de tamaño.

¿Qué es HTML Canvas?

Este elemento se utiliza para dubujar gráficos, sobre la marcha, mediante JavaScript.
Este elemento es sólo un contenedor de gráficos. Debes usar JavaScript para dibujar los gráficos.
canvas tiene varios métodos para dibujar trazados, cuadros, texto y agregar imágenes.

Ejemplos de canvas

Un canvas es un área rectangular en una página HTML.
De forma predeterminada, un canvas no tiene borde ni contenido.

El marcado se ve así:

<canvas id="myCanvas" width="200" height="100"></canvas>

Nota: Especifique siempre un atributo id (al que se hará referencia en un script) y un atritubo
width y height para definir el tamaño del canvas

Ejemplo
<canvas id="myCanvas" width="200" height="100" style="border:1px solid #000000;">
</canvas>

Agregar un JavaScript

Después de crear el área del lienzo rectangular, debes agregar JavaScript para realizar el dibujo.

Aquí hay unos ejemplos:

Dibuja una línea

<script>
var c = document.getElementById("myCanvas");
var ctx = c.getContext("2d");
ctx.moveTo(0, 0);
ctx.lineTo(200, 100);
ctx.stroke();
</script>
  1. var c = document.getElementById ("myCanvas");
    • Selecciona el elemento HTML con el atributo 'id' igual a "myCanvas" y lo alamacena en la
      variable 'c'. Este elemento debería se un elemento '<canvas> en tu HTML.
  2. var ctx = c.getContext("2d");
    • Obtiene el contexto de renderizadp 2D del canvas. El contexto 2D es esencial para
      dibujar en el canvas. Se almacena en la variable 'ctx'.
  3. ctx.moveTo(0, 0);
    • Establece el punto de inicio para el trazado de la línea en las coordenadas (0,0) dentro del canvas.
  4. ctx.LineTo(200, 100);
    • Crea una línea desde el punto de inicio (0,0) hasta las coordenadas (200,100).
  5. ctx.stroke();
    • Dibuja la línea en el canvas utilizando el estilo de trazo actual. En este caso,
      estás utilizando el método 'stroke' para trazar la línea.

En resumen, este código configura un canvas, obtiene el contexto 2D para ese canvas,
mueve el "lapiz" a un punto de inicio (0,0), traza una línea hasta un punto final (200,100), y finalmente
dibuja esa línea en el canvas. El resultado será una línea diagonal en el canvas desde
el punto (0,0) hasta el punto (200,100).

Dibuja un circulo

<script>
var c = document.getElementById("myCanvas");
var ctx = c.getContext("2d");
ctx.beginPath();
ctx.arc(95, 50, 40, 0, 2 * Math.PI);
ctx.stroke();
</script>
  1. var c = document.getElementById ("myCanvas");
    • Al igual que en el primer ejemplo, selecciona el elemento HTML con el atributo 'id'
      igual a "myCanvas" y lo almacena en la variable 'c'.
  2. var ctx = c.getContext ("2d");
    • Obtiene el contexto de renderizado 2D del canvas y lo almacena en la variable 'ctx'
  3. ctx.beginPath();
    • Inicia un nuevo trazado. Esto es importante antes de comenzar a dibujar formas como circulos, para
      que el trazado anterior (si lo hay) no afecte la nueva forma.
  4. ctx.arc(95, 50, 40, 0, 2 * Math.PI);
    • Dibuja un arco, los parámetros son :
      • '(95, 50)': Las coordenadas del centro del circulo.
      • '40': El radio del circulo.
      • '0': El ángulo inicila del arco en radianes (En este caso, comienza desde el lado
        derecho del círculo)
      • '2 * Math.PI: El ángulo final del arco en radianes (En este caso, se completa un
        círculo completo de 360 grados)
  5. ctx.stroke;
    • Dibuja el contorno del círculo en el canvas utilizando el estilo de trazo actual.

En resumen, este código configura un canvas, obtiene el contexto 2D para ese canvas,
inicia un nuevo trazado, dibuja un círculo con centro en (95,50), radio de 40 unidades y luego dibuja el
contorno del círculo en el canvas. El resultado será un círculo en el canvas.
